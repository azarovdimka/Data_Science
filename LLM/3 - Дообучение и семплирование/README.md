# 🤖 Дообучение GPT-2 и исследование методов семплирования

> Проект по fine-tuning русскоязычной модели GPT-2 на корпусе новостей Lenta.ru с последующим исследованием различных стратегий генерации текста

## 📋 Описание проекта

Данный проект демонстрирует полный цикл работы с языковой моделью:
1. Подготовка и токенизация данных
2. Fine-tuning предобученной модели
3. Оценка качества обучения
4. Экспериментирование с различными методами семплирования

## 🎯 Цели проекта

- **Дообучить** модель GPT-2 на русскоязычном корпусе новостей
- **Исследовать** влияние различных параметров генерации на качество текста
- **Определить** оптимальные настройки для генерации связного и осмысленного текста

## 📊 Данные

**Датасет:** Lenta.ru News Dataset
- **Источник:** Корпус новостей с сайта Lenta.ru
- **Размер обучающей выборки:** 15,000 текстов
- **Разделение:** 85% train / 15% validation
- **Средняя длина текста:** ~179 слов (~359 токенов)

## 🔧 Технологический стек

- **Модель:** `ai-forever/rugpt3small_based_on_gpt2` (125M параметров)
- **Фреймворк:** Hugging Face Transformers
- **Библиотеки:** PyTorch, Pandas, NumPy, Matplotlib
- **Оптимизатор:** AdamW
- **Scheduler:** Cosine

## 🎲 Исследование методов семплирования

### Протестированные методы

#### 1. **Greedy Decoding** (`do_sample=False`)
- Всегда выбирает самый вероятный токен
- Детерминированный результат
- **Результат:** Повторяющиеся фразы, низкая креативность

#### 2. **Temperature Sampling**
- **Temperature = 0.5:** Консервативная генерация, хорошая связность
- **Temperature = 1.5:** Высокая креативность, но появляются ошибки
- **Вывод:** Оптимальная температура ≈ **0.5**

#### 3. **Top-K Sampling**
- **K = 10:** Ограниченное разнообразие
- **K = 50:** Умеренное разнообразие
- **K = 100:** Наилучший баланс связности и креативности
- **Вывод:** Оптимальное значение K ≈ **100**

#### 4. **Top-P (Nucleus) Sampling**
- **P = 0.8:** Меньше связности
- **P = 0.95:** Более связный и осмысленный текст
- **Вывод:** Оптимальное значение P ≈ **0.95**

#### 5. **Min-P Sampling**
- **min_p = 0.05:** Хорошая грамотность
- **min_p = 0.1:** Баланс разнообразия и качества
- **min_p = 0.2:** Наилучший результат
- **Вывод:** Оптимальное значение min_p ≈ **0.2**

#### 6. **Beam Search**
- **num_beams = 2:** Быстрая генерация, хорошие результаты
- **num_beams = 4:** Много повторов
- **num_beams = 8:** Много повторов
- **Вывод:** Оптимальное значение num_beams = **2**

## 🚀 Как использовать

### Загрузка модели

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained("./output/final_model")
tokenizer = AutoTokenizer.from_pretrained("./output/final_model")
```

### Генерация текста

```python
from transformers import pipeline

generator = pipeline('text-generation', model=model, tokenizer=tokenizer)

output = generator(
    "Мы живем в удивительное время",
    max_new_tokens=50,
    temperature=0.5,
    top_k=100,
    top_p=0.95,
    do_sample=True
)

print(output[0]['generated_text'])
```
