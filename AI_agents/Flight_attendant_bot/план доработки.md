### Шаг 1. Выбрать инструктивную модель эмбеддингов
**Варианты:**
- `ai-forever/FRIDA` — лучшая для русского, префиксы `search_document:` / `search_query:`

**Важно:** Это **retrieval-модель**, а не LLM. Её не дообучают через QLoRA — QLoRA это для генеративных моделей (GPT, LLaMA). Если нужно адаптировать под авиацию, используется **fine-tuning retrieval** (MultipleNegativesRankingLoss на парах вопрос-ответ), но это отдельная большая задача. **Для текущего ДЗ берём готовую модель без дообучения.**

### Шаг 2. Разбить базу знаний на чанки (_structural chunking_)
**Порядок критичен: сначала чанкинг, потом векторизация.**

**Почему:** Вектор — это сжатое числовое представление _целого текста_. Если сначала векторизовать 1000 слов, а потом «разрезать» вектор — получится бессмыслица. Каждый чанк должен быть **самостоятельным семантическим блоком** со своим вектором.

**Для авиационных регламентов (FAQ, нумерованные пункты):**
```python
def chunk_by_structure(text: str) -> List[str]:
    # Разбиваем по номерам вопросов: "1. ", "2)", "15. Текст"
    pattern = r'(?=\n\s*\d+[\.)\s]+\s*[А-ЯA-Z])'
    chunks = re.split(pattern, text)
    return [c.strip() for c in chunks if len(c.strip()) > 30]
```

**Результат:** один чанк = один вопрос-ответ или один регламентный пункт.

### Шаг 3. Извлечь метаданные и метки (автоматически)
Для каждого чанка извлекаем метки **до** загрузки в Qdrant:

```python
# Метаданные для payload
metadata = {
    "text": chunk_text,                    # исходный текст

```

**Автоматическое извлечение меток** — это NER/регулярки на этапе препроцессинга (как в RAG 2, раздел 4). Не в Qdrant, а в Python перед `upsert`.

### Шаг 4. Векторизовать чанки инструктивной моделью **с префиксами**

### Шаг 5. Перенести в персистентную Qdrant с payload
```python
client = QdrantClient(path="./qdrant_storage")  # ← на диск, не в память

client.create_collection(
    collection_name="aviator_regulations",
    vectors_config=VectorParams(size=1024, distance=Distance.COSINE)
)

points = [
    PointStruct(
        id=chunk_id,
        vector=embeddings[i].tolist(),
        payload={
            "text": chunk_texts[i],
            "parent_id": metadata["parent_id"],
            "aircraft_types": metadata["aircraft_types"],
            "procedure_categories": metadata["procedure_categories"],
            "word_count": metadata["word_count"],
        }
    )
    for i in range(len(chunks))
]

client.upsert(collection_name="aviator_regulations", points=points, wait=True)
```

**Payload** — это не «колонки в БД» в классическом смысле, а JSON-метаданные к каждому вектору. Qdrant хранит их вместе с вектором и позволяет фильтровать по ним при поиске.
