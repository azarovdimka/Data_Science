# 🏨 Детекция неполных описаний комнат для Т-Путешествия

## 📋 Описание задачи

### Контекст
Сервис **Т-Путешествия** ежедневно обрабатывает множество предложений по бронированию отелей от разных операторов (Booking.com, TravelLine, Expedia и др.). Для того чтобы ожидания клиентов были оправданы, необходимо сопоставлять эти предложения с эталонным описанием комнаты — **мастер-комнатой**.

### Проблема
Часто поставщики предоставляют **неполные или неоднозначные описания комнат**, из-за чего:
- Невозможно однозначно идентифицировать комнату даже вручную
- Увеличивается число заданий в краудсорсинге
- Растут операционные издержки
- Снижается качество сопоставления

### Цель
Научиться **заранее предсказывать**, что в описании комнаты недостаточно данных для сопоставления, чтобы:
- Минимизировать поток таких заданий на ручную обработку
- Запрашивать дополнительные данные у операторов
- Достичь максимальной точности информации

---

## 🎯 Постановка ML-задачи

### Тип задачи
**Бинарная классификация**

### Целевая переменная
- `is_incomplete = 1` — описание неполное (недостаточно данных для матчинга)
- `is_incomplete = 0` — описание полное (достаточно данных)

### Метрики качества
1. **F1-Score** — баланс между Precision и Recall
2. **ROC-AUC** — общая способность модели различать классы
3. **Precision** — минимизация ложных срабатываний
4. **Recall** — максимизация обнаружения неполных описаний

## 🔧 Feature Engineering

### 1. Признаки на основе пропусков
Сосчитали количество признаков с пропусками вывели в отдельный признак

```python
missing_count = количество NaN в [area, bed_type, view, max_guests, floor, n_rooms, rating]
```

**Логика**: Чем больше пропусков, тем выше вероятность неполного описания.

### 2. Признаки длины текстовых полей

Замерили длину текстовых описаний.

```python
room_class_len = len(room_class)
description_len = len(description)
amenities_count = количество элементов в списке удобств
```

**Логика**: Короткие или пустые описания указывают на неполноту данных.

### 3. Бинарные признаки наличия данных

Указаны ли данные в соответсвующих колонках - раздали бинарные признаки

```python
has_area = 1 если area не NaN, иначе 0
has_bed_type = 1 если bed_type не пустая строка, иначе 0
has_view = 1 если view не пустая строка, иначе 0
has_max_guests = 1 если max_guests не NaN, иначе 0
has_floor = 1 если floor не NaN, иначе 0
has_n_rooms = 1 если n_rooms не NaN, иначе 0
has_rating = 1 если rating не NaN, иначе 0
has_description = 1 если description_len > 0, иначе 0
has_room_class = 1 если room_class_len > 0, иначе 0
```

**Логика**: Явное указание наличия/отсутствия каждого ключевого поля помогает модели выявлять паттерны.

### 4. Заполнение пропусков

После создания дополнительных признаков-маркеров, пропуски в исходных числовых полях заполняются медианой:

```python
area = area.fillna(median)
max_guests = max_guests.fillna(median)
floor = floor.fillna(median)
n_rooms = n_rooms.fillna(median)
rating = rating.fillna(median)
```

Таким образом, мы увеличили количество признаков. Наличие и тех и других признаков полезно для моделей (и с заполненной информацией, и пометки о том, что там не было информации - модель сама выберет важные признаки. А с пропусками модели работать не могут.)
