# **🛒 Прогнозирование покупок в интернет-магазине**

**Машинное обучение для предсказания вероятности покупки клиента в течение 90 дней**

## 📋 **Описание проекта**

Интернет-магазин собирает историю покупателей и проводит маркетинговые рассылки. Для оптимизации кампаний необходимо выделить клиентов, готовых совершить покупку в ближайшее время.

- **Задача:** Предсказать вероятность покупки клиента в течение 90 дней на основе истории покупок и взаимодействия с рассылками.

- **Цель:** Разработать ML-модель для классификации клиентов по вероятности совершения покупки в течение 90 дней.

## ⚠️ **Основные проблемы**

- **Сильный дисбаланс классов:** 98% клиентов не совершают покупку (соотношение 98:2)

- **Слабые корреляции:** Максимальная корреляция признаков с целевой переменной = 0.15

- **Сложная структура данных:** 5 таблиц с разной гранулярностью требуют агрегации

## 📊 **Данные**

- **Источник:** 354 тыс. записей из 5 таблиц

- **После агрегации:** 49,849 уникальных клиентов

- **Признаки:** 70+ созданных признаков (RFM-метрики, категории товаров, взаимодействие с кампаниями)

## 🛠 **Технологический стек**

● `Python 3.x` 

● `Pandas`, `NumPy` — обработка данных

● `Scikit-learn` — ML-пайплайны, метрики

● `LightGBM`, `CatBoost`, `LogisticRegression` — модели

● `Dython` — анализ корреляций

● `KMeans` — сегментация клиентов

## 🚀 **Что было сделано**

**1. Предобработка данных**

- Агрегация 5 таблиц к уровню client_id

- Удаление дубликатов и выбросов

- Обработка пропусков через Pipeline

**2. Feature Engineering**
Создано 70+ признаков:

- RFM-метрики (Recency, Frequency, Monetary)

- Иерархические категории товаров (4 уровня)

- Признаки взаимодействия с кампаниями

- Временные признаки (сезонность, давность)

- Сегментация клиентов (KMeans): активные (6.8% конверсия) vs неактивные (1.6%)

**3. Борьба с дисбалансом**

- Балансировка классов (class_weight='balanced')

- Оптимизация по метрике PR-AUC

- Тестирование SMOTE (не дало улучшений)

**4. Обучение моделей**

- Обучено 3 модели: LogisticRegression, LightGBM, CatBoost

- GridSearchCV для подбора гиперпараметров

- Отбор топ-30 признаков по важности

## 📈 **Результаты**

| **PR-AUC** | 0.13 | 0.09 | 0.09 | +350% (от 0.02) |
| **ROC-AUC** | 0.82 | 0.75 | 0.76 | +52% (от 0.50) |


## ✅ **Достижения**

- **Модель превосходит baseline * 4.5 раза по PR-AUC**

- **ROC-AUC 0.76** — хорошая способность ранжировать клиентов

- **Стабильная генерализация:** падение Train→Test всего -7% по ROC-AUC

- **Production-ready:** Pipeline готов к интеграции

## 💡  **Ключевые инсайты**

- **Категории товаров** (top_category_4) — сильнейший предиктор 4-й уровень вложенности (корреляция 0.15)

- **Активность важнее денег:** active_days (0.11) > total_amount (0.03)

- **RFM работает частично:** Recency и Frequency важны, Monetary — нет

- **Mobile push эффективнее** email для конверсии
